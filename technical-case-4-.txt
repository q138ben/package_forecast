As a data scientist you would have to contribute by providing a lot of
different forecasts that are used in our daily operation. In this case study we want to
see how you approach and solve such problems.
# Task
Our stakeholders want to optimize operations at three locations (A, B, and C). They
need to have estimated daily number of packages that each location is going to receive
for the upcoming 30 days. They would like to receive these numbers in as automated way
as possible (preferably through API calls).
You may find some historical data for these locations in data.csv file. We are asking
you to help us analyze the data, create forecasts and expose them.
# Breakdown of tasks (our evaluation points):
* explore the data (feel free to modify and enrich it if needed)
* create forecasting model(s) for locations A and B (C would be a bonus):
    * explain your choice of the model(s) and what other models you have considered
    * evaluate your model performance and explain the choice of metrics
    * discuss the parameters and configuration used
* generate forecast:
    * number of packages at each location for 30 days in the future (daily values)
    * bonus: add values/metrics that would explain how certain/uncertain each forecast
             is
* create an API:
    * expose the results through some API implementation (assume that no strict
      requirements for the output were provided)
    * bonus: deploy API to cloud (we are using GCP, but you may choose any other
      cloud provider)
* structure the project and code so that it would be easy to read and understand
# Notes:
* We are mostly using Python in our daily work, but you are free to use any other
  language (as long as it is rather widely used) if you prefer.
* Try to cover as many points on list above as possible. If you get stuck or some
  parts become too complex do not spend too much time on trying to solve everything.
  You can instead describe how the problem would be solved when given more time.
* Add comments and explanations where relevant. It should be easy for us to go through
  the solution and understand your reasoning.
*Note that we prefer that you avoid using AI to solve this problem. So the code should be
written in way that does not look like AI.
